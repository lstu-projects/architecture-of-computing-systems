# Система управления заказами интернет-магазина

## Описание проекта

Информационная система для управления заказами интернет-магазина с использованием **паттерна проектирования State (
Состояние)**.

Система позволяет:

- Отслеживать жизненный цикл заказа через последовательность статусов
- Автоматически оповещать покупателя при каждом изменении статуса
- Оповещать кладовщика при разрешении отгрузки
- Оповещать курьера при разрешении доставки
- Блокировать недопустимые переходы между статусами

---

## Цель работы

Изучить и применить на практике **поведенческий паттерн проектирования State** для управления поведением объекта в
зависимости от его внутреннего состояния и обеспечения чистых переходов между состояниями без условных операторов.

---

## Структура проекта

```
lab-03-behavioural-patterns/
│
├── state/
│   ├── OrderState.java               # Состояние (State) - интерфейс с default-методами
│   ├── AcceptedState.java            # Конкретное состояние - Принят
│   ├── ProcessingState.java          # Конкретное состояние - В обработке
│   ├── PaidState.java                # Конкретное состояние - Оплачен
│   ├── ShipmentApprovedState.java    # Конкретное состояние - Разрешена отгрузка
│   ├── DeliveryApprovedState.java    # Конкретное состояние - Разрешена доставка
│   ├── DeliveredState.java           # Конкретное состояние - Доставлен
│   └── CancelledState.java           # Конкретное состояние - Отменён
│
├── context/
│   └── Order.java                    # Контекст (Context) - делегирует вызовы состоянию
│
├── observer/
│   ├── Customer.java                 # Покупатель - оповещается при каждом переходе
│   ├── Warehousekeeper.java          # Кладовщик - оповещается при отгрузке
│   └── Courier.java                  # Курьер - оповещается при доставке
│
├── Main.java                         # Демонстрация работы системы
```
